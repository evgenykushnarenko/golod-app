# AGENTS.md

## Назначение репозитория

Репозиторий содержит исходный код PWA-приложения "Счетчик голодания" — инструмента для отслеживания времени голодовки с визуализацией прогресса и мотивационными фразами.

## Основные файлы

- **index.html** — исходная HTML-точка входа для Vite
- **public/manifest.json** — конфигурация PWA
- **public/service-worker.js** — service worker
- **src/app.js** — основной JavaScript код (таймер, прогресс, настройки, история, уведомления)
- **src/motivation.js** — мотивационные фразы
- **src/styles.css** — стили на нативном CSS
- **vite.config.js** — конфигурация сборки
- **dist/** — результат сборки, не источник правок

`tailwind.config.js` и `postcss.config.js` присутствуют в репозитории, но текущий UI на них не опирается.

## Обязательные проверки

После любых изменений:

1. Проверить сборку: `npm run build`
2. Убедиться, что в `dist/` корректно лежат `index.html`, `manifest.json`, `service-worker.js` и свежие ассеты
3. Если менялся UI/PWA, проверить отсутствие ошибок в консоли браузера

## Ограничения и риски

- Значения по умолчанию для старта и финиша заданы в `src/app.js`, но могут быть переопределены через `localStorage`
- Приложение использует локальное время браузера и не синхронизируется с сервером
- Уведомления зависят от разрешения браузера и корректной регистрации service worker
- Состояние темы, истории и пользовательских фраз хранится только локально

## Правила изменений

- Коммиты на русском языке
- Изменения в конфигурации должны сопровождаться обновлением документации
- Добавление новых зависимостей — только после проверки необходимости
- После каждого логически завершенного этапа — `git push`

## Зависимости

- vite (сборка)
- tailwindcss / postcss / autoprefixer (legacy-конфигурация, сейчас не участвует в UI)
- chart.js (зависимость осталась в проекте, но текущий экран её не использует)

## Базовая модель работы

1. Изменения делаем локально в `src/`
2. Для PWA-части при необходимости меняем `public/`
3. Проверяем через `npm run build`
4. Публикуем собранный `dist/`, а не копируем сборочные артефакты обратно в исходный `index.html`
